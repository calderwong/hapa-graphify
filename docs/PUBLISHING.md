# Publishing

This repo can be published to GitHub while keeping local-private graph data out of git.

## Publish Boundary

Commit:

- Source code under `hapa_graphify/`.
- Tests and fixtures.
- Docs and schema files.
- UI source.
- Public-safe source registry with environment-variable roots.
- Public-safe GitHub Pages demo data.

Do not commit:

- Full reports in `reports/*.json`.
- Run ledgers in `runs/`.
- `graphify-out/` caches.
- SQLite DBs, WAL/SHM files, token files, `.env`, raw private media, or private exports.

## GitHub Pages

Pages is served from `docs/`.

The static demo lives at:

```text
docs/demo/index.html
```

The demo data is generated by:

```bash
.venv/bin/python scripts/build_pages_demo.py
```

The generator reads local compact map sidecars and writes `docs/demo-data/atlas.json` with private paths, record keys, source files, and samples omitted.

## Suggested Publish Flow

```bash
git init
git add .
git commit -m "Document and publish Hapa Graphify"
gh repo create hapa-graphify --public --source=. --remote=origin --push
gh api --method POST repos/<owner>/hapa-graphify/pages -f source[branch]=main -f source[path]=/docs
```

If Pages already exists:

```bash
gh api --method PUT repos/<owner>/hapa-graphify/pages -f source[branch]=main -f source[path]=/docs
```

Expected URL:

```text
https://<owner>.github.io/hapa-graphify/
```

## Pre-Publish Checks

```bash
.venv/bin/python -m compileall -q hapa_graphify
.venv/bin/python -m unittest discover -s tests
.venv/bin/python scripts/build_pages_demo.py
rg -n "/Users/<your-name>|/Volumes/|BEGIN .*PRIVATE KEY|ghp_|sk-" README.md AGENTS.md .env.example docs config hapa-node.json scripts hapa_graphify
```

The final `rg` should not reveal private local paths or credential-like literals in committed docs/source. Policy mentions of `.env`, `.node_token`, WAL/SHM files, and private media are expected in docs and manifests.
